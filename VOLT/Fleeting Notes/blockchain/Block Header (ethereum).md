
### The Block Header (The Context)

The header contains metadata that validates the block’s placement in history.

- **Parent Root / Parent Hash:** The cryptographic link to the previous block, keeping the chain unbroken.
    
- **State Root:** The cryptographic hash of the entire global ledger _after_ this block’s transactions are run.
    
- **Slot & Epoch:** The precise timeline markers showing exactly when the block was proposed.
    
- **Proposer Index:** The numerical ID of the validator who was chosen to build the block.
